Custom Persons#

Creating, deleting and reading person data#

To create a new person in a project, delete a person from a project or retrieve a list of persons for a given project or the details of a specific person, you can use the vidrovr.resources.Person object along with the PersonModel data object:

pydantic model src.vidrovr.resources.custom_persons.PersonModel#

Model of a person.

Parameters

  • id (str) – ID of the person

  • name (str) – Name of the person

  • creation_date (str) – Creation date of the person

  • is_active (bool) – Indicates if the person is active or not

  • num_examples (int) – Number of examples of the person

  • num_reviewed (int) – Number of examples of the peson that have been reviewed

  • num_unreviewed (int) – Number of example of the person that have not been reviewed

Creating, reading and updating person examples#

Model of a person example.

  • href (str) – URL of the person

src.vidrovr.resources.custom_persons.PersonExample.read(project_id: str, person_id: str, example_id: Optional[str] = None)#

Returns either a single person or an array of person examples identifiers, paginated, and filtered by status, which can be one of three values: accepted, rejected or candidate. The first value of page is always 0.

Parameters

  • project_id (str) – ID of the project containing the person examples

  • person_id (str) – ID of the person in the project

  • example_id (str) – ID of the specific example to pull details. Defaults to None to return the array of all examples.

Returns

A single person or an array of person examples

Return type

PersonExampleData
